Off the top of my head I could come up with 3 different options:

  1. One-vs-All (OvA) / One-vs-Rest (OvR)
  2. One-vs-One (OvO)
  3. Softmax
1. and 2. are not really "generalizations" in a mathematical sense. Here, you just train multiple binary logistic regression classifiers.

OvA:


In OvA, you train C classifiers, where C is the number of unique class labels.

Let's assume we have 3 classes, then you'd train 3 logistic regression classifiers

  • class 1 vs class 2 & 3
  • class 2 vs class 1 & 3
  • class 3 vs class 1 & 2

Then, when you make a prediction, you take the "argmax," that is, the predicted class is the class with the highest probability. The downside though is that the class probabilities are not "meaningful" (i.e,. the don't sum up to one). For example, you could end up with something like

  • class 1 vs class 2 & 3 ---> 0.78
  • class 2 vs class 1 & 3 ---> 0.67
  • class 3 vs class 1 & 2 ---> 0.91
=> predict class 3

or so. This is okay though if you are only interested in the class labels. If you want to have "meaningful" probabilities, use softmax.


OvO:


In OvO, you train C (C − 1) / 2 classifiers, where C is the number of unique class labels. For example,

  • class 1 vs class 2 => class 1
  • class 1 vs  class 3 => class 2
  • class 2 vs class 3 => class 2
=> predict class 2

Then, you use e.g., a majority voting scheme to make the prediction.


Softmax:


Softmax is more interesting, here, you replace the logistic sigmoid



function by the softmax function



With softmax, we can get meaningful class-probabilities in multi-class settings (sometimes also called multinomial logistic regression). In softmax, the probability of a particular sample with net input  belongs to the ith class can be computed with a normalization term in the denominator that is the sum of all  linear functions. After you computed the class-membership probabilities, you take the argmax (like in OvA).

Answered Sep 11, 2015
If your response variable is ordinal in nature (Example: Low - Medium - High) then it is easy to understand how binary logistic regression is extended to these cases. In a nutshell: you use a different intercept for each of the levels while retaining the same coefficients. Every statistical package has this extension, such as polr() in R and proc logistic in SAS.

If your response variable is categorical, i.e. there is no inherent ordering, then the best thing to do is to use Multinomial Logistic Regression (MNL). For example, if the response variable represents choices (such as which mode of transport is likely to be selected by a certain population), then each choice has a probability of selection calculated within the choice set. In binomial logistic regression, if the predicted probability is P, then P and (1-P) represent the probability of occurrence and non-occurrence. In multinomial logistic regression, if there are 'n' choices, then you get P1, P2, P3... Pn where they sum up to 1.

MNL theory is vast and well developed. There are a lot of variations (weighted models, nested models, cross-nested models etc.) that cover all kinds of situations. As a branch of learning it is generally called Discrete Choice Models.
